Control de versiones • Git

¿Qué es un commit y por qué es importante?

Un commit es un registro (snapshot) de los cambios realizados en el código. Incluye qué se cambió, quién lo hizo, cuándo y un mensaje breve que explica el motivo.

  • Historial claro: permite ver la evolución del proyecto.
  • Reversibilidad: facilita deshacer cambios o identificar cuándo apareció un error.
  • Colaboración: cada contribución queda registrada y se puede revisar.
  • Automatización y trazabilidad: dispara CI/CD y deja información sobre el autor y la intención.
  • Mejores revisiones: commits pequeños y descriptivos simplifican code review.

# ejemplo rápido
git add .
git commit -m "Arregla validación de enteros en sumaEnteros"
Consejo: escribe mensajes claros y atómicos; piensa cada commit como una unidad lógica de cambio.
